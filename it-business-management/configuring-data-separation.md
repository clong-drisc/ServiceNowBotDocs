# Configuring Data Separation

As an administrator, you can install Data Separation and assign the Data Separation roles to the required users.

## Configuration workflow

A Data Separation administrator can define Data Separation hierarchy, enable entities, and map user groups for Data Separation.

## Guided setup to configure Data Separation

Guided setup provides a sequence of tasks to help you with the required configuration on your ServiceNow instances. After installing Data Separation, navigate to **All** &gt; **Data Separation** &gt; **Guided Setup** to configure Data Separation.

-   **[Install Data Separation](../task/install-data-separation.md)**  
You can install the Data Separation application \(sn\_ds\) if you have the admin role.
-   **[Assign roles for managing Data Separation](../task/assign-roles-for-managing-data-separation.md)**  
Assign Data Separation roles to user groups and to individual users based on user activities and responsibilities.
-   **[Configure a lens for enabling Data Separation](../task/configure-lens-for-data-separation.md)**  
Configure a data separation hierarchy for enabling Data Separation using a lens hierarchy and its leaf node.
-   **[Enable entities for Data Separation](../task/map-entities-for-data-separation.md)**  
Enable Data Separation on select entities as needed.
-   **[Map a user group for Data Separation](../task/map-groups-for-data-separation.md)**  
Map a user group for Data Separation to restrict data only to the specified users of a group and sub-groups.
-   **[Synchronize Data Separation](../task/synchronize-data-separation.md)**  
Synchronize Data Separation when the configuration is changed in the lens, supported entities, or entity-group mappings tables.
