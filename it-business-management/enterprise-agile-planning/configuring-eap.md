# Configuring Enterprise Agile Planning

Activate a configuration, add teams to your Agile structure, and define your planning calendars so that your team can get started with Backlog management and PI planning in EAP.

## Configuration overview

The following tasks require the EAP admin \(sn\_apw\_advanced.eap\_admin\) role.

1.  [Install Strategic Planning](../../alignment-planner-workspace/task/install-alignment-planner-workspace.md).

    Enterprise Agile Planning is available within the Strategic Planning workspace.

2.  Assign the EAP roles to users. See [Components installed with Enterprise Agile Planning](../reference/components-installed-with-enterprise-agile-planning.md).
3.  Learn about EAP configurations, team level hierarchy, and work item type rollup.

    See [Agile configurations in EAP](agile-configurations-in-eap.md).

4.  \(Optional\) [Create or update a configuration in EAP](../task/create-eap-configuration.md).
5.  [Activate an EAP configuration](../task/activate-eap-configuration.md)
6.  Define the agile structure by adding teams to your EAP configuration. See [Define agile structure in EAP](../task/define-agile-structure-in-eap.md).
7.  [Configure dashboards for teams in your Agile structure](configuring-eap-dashboard.md).

The following tasks require the system admin role.

1.  \(Optional\) Create additional planning calendar types. See [Create a planning calendar in EAP](../task/create-planning-calendar-type-in-eap.md).
2.  [Create calendar entries for iterations in EAP](../task/create-calendar-entries-in-eap.md).
3.  \(Optional\) If you want a different type of team or work item than those available by default, you can create custom types. See [Create a custom team type in EAP](../task/create-custom-team-type-in-eap.md) and [Create a custom work item type in EAP](../task/create-custom-work-item-type-in-eap.md).
