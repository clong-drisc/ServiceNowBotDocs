# Process Mining reference

Reference topics provide additional information about the lists and forms that you use to configure and administer Process Mining.

-   **[Components installed with Process Mining](../reference/components-installed.md#)**  
Several types of components are installed with activation of the com.sn\_po plugin, including tables, user roles, and scheduled jobs.
-   **[Process Mining key terms](../reference/key-terms.md)**  
Key terms used in Process Mining.
-   **[Domain separation and Process Mining](domain-separation-process-optimization.md)**  
Domain separation is supported in Process Mining. Domain separation enables you to separate data, processes, and administrative tasks into logical groupings called domains. You can then control several aspects of this separation, including which users can see and access data.
-   **[Project definition form](../reference/new-project-def.md)**  
Use the Project definition form to create a project.
-   **[Create new process start/end condition](../reference/process-start-condition.md)**  
Crop your process by setting the new process start condition and new process end condition. This configuration defines which part of the process should be included in the Process Mining project and made available for analysis. The Create new process start and end condition forms have the same fields.
-   **[Activity definition form](../reference/activity-def.md)**  
Use the Activity Definition form to report on steps that occur within your business process.
-   **[Automated finding definition form from Finding Builder](../reference/auto-finding-form-builder.md)**  
Use the Create Improvement opportunity definition form to create an automated finding definition from the Finding Builder.
-   **[Automation finding definition form from Classic view](../reference/auto-find-form-classic.md)**  
Use the Automated Finding definition form to create a finding definition from the Classic view.
-   **[Rule-based finding definition form from Finding Builder](../reference/finding-definition-form.md)**  
Use the Create Improvement opportunity definition form to create a finding definition from the Finding Builder.
-   **[Rule-based finding definition form from Classic view](../reference/finding-def-form.md)**  
Use the Finding Definition form to configure rule-based finding definitions to deliver discovered insights to your Summary and Insights page.
-   **[Finding rule form](../reference/finding-rule-form.md)**  
Use the Finding Rule form to configure finding rules for your finding definition. Finding rules define what record criteria triggers your finding definitions.
-   **[Finding constraint form](../reference/finding-constraint-form.md)**  
Use the Finding constraint form to configure finding constraints for your finding definition. In addition to finding rules, finding constraints further narrow down the scope of your finding definitions using metrics like duration and relation constraints.
