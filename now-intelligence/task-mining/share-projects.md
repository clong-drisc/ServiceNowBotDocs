# Share a Task Mining analysis

Determine which process owners have access to the project's analysis.

## Before you begin

Role required: sn\_tm\_core.analyst, sn\_tm\_core.power\_user, sn\_tm\_core.admin

## Procedure

1.  Navigate to **Workspaces** &gt; **Task Mining Workspace**.

2.  Select the project.

3.  Select **Shared**.

4.  If the project has already been shared with some users, determine whether you still want to grant those users permissions for the project's analysis.

    -   To retain a user and the user's current permissions, leave the user name in the list.
    -   To change a user's current permissions, select the user name, modify the permissions in the Permissions dialog box, and select **Save**.
    -   To remove all sharing permissions for a user, select the check box for the user and select **Remove**.
5.  Determine new users with whom you want to share the project's analysis.

    1.  Select **Share**.

    2.  In the **User** field, select the user with whom you want to share the analysis.

    3.  Select whether to grant Edit permissions to the user.

    4.  Select **Save**.

6.  Repeat Step [5](share-projects.md#sharestep) for all users you want to grant permission to access the project's analysis.
