# Access control for incidents

To help separate Operational Technology \(OT\) and Information Technology \(IT\) data, only OT users can view OT incidents.

The following table describes the roles and permissions for the users that have the Operational Technology Incident Management roles.

|Role|Permissions|
|----|-----------|
|sn\_ot\_incident\_write|Can create, edit, and read OT incidents.|
|sn\_ot\_incident\_read|Can only read OT incidents.|
|sn\_ot\_incident\_admin|Can create, view, edit, and delete incident records for any equipment model entity.|

For more information about access control rules, see [Access control rules in application administration apps](https://www.servicenow.com/docs/access?context=ACL-access-checks&version=yokohama&pubname=yokohama-application-development&ft:locale=en-US).
