# Configuring data inputs for Health Log Analytics manually

Configure the data input process manually in Health Log Analytics. Data input configuration is an essential step in setting up the Health Log Analytics application.

-   **[Configure Rsyslog, Filebeat, or Winlogbeat data inputs](../task/hla-data-input-rsyslog-beats.md)**  
Configure a data input for streaming log messages to your ServiceNow instance using an Rsyslog, Filebeat, or Winlogbeat agent.
-   **[Configure Splunk data inputs](../task/hla-data-input-splunk.md)**  
Configure a data input for streaming log messages to your ServiceNow instance using a Splunk heavy forwarder.
-   **[Configure Splunk Polling data inputs](../task/hla-data-input-splunk-polling.md)**  
Configure a data input that periodically pulls log data from Splunk by using a query.
-   **[Configure Elasticsearch data inputs](../task/hla-data-input-elastic.md)**  
Configure a data input for streaming log data from Elasticsearch indices to your ServiceNow instance.
-   **[Configure TCP data inputs](../task/hla-data-input-tcp.md)**  
Configure a data input for sending raw log messages to your ServiceNow instance directly over a TCP/SSL socket.
-   **[Configure UDP data inputs](../task/hla-data-input-udp.md)**  
Configure a data input for sending raw log messages to your ServiceNow instance directly over a UDP socket.
-   **[Configure GCP PubSub data inputs](../task/hla-data-input-gcp-pubsub.md)**  
Configure a data input for receiving log messages that are published to a Google Cloud Pub/Sub topic and streaming them to your ServiceNow instance.
-   **[Configure MID Server data inputs](../task/hla-data-input-mid-server.md)**  
Configure a data input for collecting and streaming MID Server log messages to your ServiceNow instance.
-   **[Configure Amazon CloudWatch data inputs](../task/hla-data-input-cloudwatch.md)**  
Configure a data input for streaming log data from Amazon CloudWatch to your ServiceNow instance.
-   **[Configure Amazon S3 data inputs](../task/hla-data-input-s3.md)**  
Configure a data input for streaming log data from Amazon S3 \(Simple Storage Service\) buckets to your ServiceNow instance.
-   **[Configure Microsoft Azure Log Analytics data inputs](../task/hla-data-input-azure.md)**  
Configure a data input for streaming log data from Microsoft Azure Log Analytics to your ServiceNow instance. The data input points the Health Log Analytics AI engine to a data source in your Microsoft Azure Log Analytics account.
-   **[Configure Microsoft Azure Event Hubs data inputs](../task/hla-data-input-event-hubs.md)**  
Configure a data input for streaming events from Microsoft Azure Event Hubs to your ServiceNow instance.
-   **[Configure Apache Kafka data inputs](../task/hla-data-input-kafka.md)**  
Configure a data input for streaming log data from Apache Kafka to your ServiceNow instance.
-   **[Configure REST API data inputs](../task/hla-data-input-rest-api.md)**  
Configure a REST API data input for streaming log data to your ServiceNow instance in JSON format.
-   **[Configure ServiceNow System Logs Retriever data inputs](../task/hla-data-input-glide-syslog.md)**  
Configure a data input for streaming log data from the ServiceNow System Log table to the Health Log Analytics AI engine \(Occultus\).
-   **[Configure Cribl data inputs](../../hardware-asset-management/task/hla-data-input-cribl.md)**  
Configure a dedicated Cribl data input to enable Health Log Analytics to process Cribl log messages streaming into your ServiceNow instance.
-   **[Configure Edge Delta data inputs](../task/hla-data-input-edge-delta.md)**  
Configure an Edge Delta data input to enable Health Log Analytics to process Edge Delta log messages streaming into your ServiceNow instance.
-   **[Configure Vector Agent data inputs](../task/hla-data-input-vector-agent.md)**  
Configure a Vector Agent data input to enable Health Log Analytics to process log messages that are streaming into your ServiceNow instance via a Vector Agent.
