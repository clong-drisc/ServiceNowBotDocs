# Configuring Instance Clone

Register your instance for cloning and create a custom clone profile.

## Configuration overview

1.  [Register an instance for cloning](configure-target-instance.md)
2.  [Register target instance \(legacy\)](t_CreateACloneTarget.md)

    Register your target instance before requesting your clone.

3.  [Create a custom clone profile](configure-clone-profile.md)
4.  [Create a custom clone profile \(legacy\)](../reference/system-profile-clone.md)

    You can create multiple clone profiles and use a reusable clone template. Clone profiles enable you to select the correct exclusions and preservers for your clone.

5.  [Create a clone preserver](create-new-clone-preserver.md)
6.  [Create a data preserver \(legacy\)](t_CreateADataPreserver.md)

    Create clone preservers to protect data on the target instance from being overwritten.

7.  [Exclude a table from cloning \(legacy\)](t_ExcludeATableFromCloning.md)
8.  [Exclude a table from cloning](exclude-a-table-from-cloning.md)

    Exclude a table to create an empty but usable table on the target instance.

9.  [Create cleanup scripts](create-cleanup-script.md#)
10. [Cleanup scripts \(lists and forms\)](create-cleanup-script.md#)

    Use cleanup scripts to automate post-clone steps or to modify data after your clone.
