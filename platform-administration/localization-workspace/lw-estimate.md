# Request translations in Localization Workspace: Estimate

Generate an informational cost estimate before you submit a translation request to your translation provider.

## Before you begin

Ensure that your localization manager has configured translation providers in Localization Workspace, with accurate costs per word \(or character\).

Role required: localization\_requestor

## About this task

**Important:**

Cost estimates are informational only. ServiceNow makes no representations or warranties as to the accuracy of this estimate. You are solely responsible for properly configuring the translation cost settings and validating the actual costs with the translation provider.

The informational cost estimate is calculated by multiplying the total number of words \(or characters\) by the cost per word \(or character\). The following conditions apply.

-   Your third-party translation provider charges you directly for translation services provided. ServiceNow has no involvement with billing or payment.
-   Each translation service provider has its own definition of how to count words for billing purposes. Your provider's word count might be different from ServiceNow's estimated word count.
-   You must input and maintain accurate costs per word in the Translation Provider record in Localization Workspace.
-   This estimate is not generated by a call to the API of your translation service.
-   The term character in the Localization Workspace interface corresponds to a Unicode code point.
-   The **Estimated total** value is rounded down to a whole number, so decimal places are not included.
-   The informational cost estimate is calculated and displayed in the currency of the translation requester's user session.
-   When you input the cost per word from your translation provider, use the base rate before any volume discount is achieved. Localization Workspace is not able to calculate volume discounts or overages, so the informational cost estimate displayed in this wizard might be different from actual charges.

Consult your provider's documentation. For details about the pricing of Google Cloud Translator Service, see [https://cloud.google.com/translate/pricing](https://cloud.google.com/translate/pricing). For details about Microsoft Azure Translator, see [https://azure.microsoft.com/en-us/pricing/details/cognitive-services/translator/](https://azure.microsoft.com/en-us/pricing/details/cognitive-services/translator/).

## Procedure

1.  Complete the first three steps \(Languages, Types, and Scope\) in the Translation request wizard, then select **Next** to proceed to **Estimate**.

    Larger translation requests take longer to generate informational cost estimates. ![The Estimate step in the Translation Request wizard in Localization Workspace, with an example estimated total highlighted.](../image/lw-estimate1.png)

2.  Review the generated informational cost estimate.

    The estimate includes total cost and word count for the overall translation request. For your convenience, the cost and word count per language are also provided.

3.  Use the **Previous** button to modify the estimate by returning to earlier steps and removing documents from your request.

4.  Select **Save**.

5.  Select **Submit** to create the bundled translation request and send to your translation providers.

    **Note:**

    -   Larger translation requests take longer to create. The process may take several minutes for very large requests.
    -   Submitting the translation request sends the documents to your third-party service providers, so you can't cancel a submitted translation request.

## What to do next

You can view a translation request record in the list on Localization Workspace's Home page.

**Note:** On the Home page, the **Rate** is displayed rounded up to the decimal place that is set in your **Display Value Currency** field. However, the calculation in Localization Workspace ignores the currency's display value. The calculation uses the actual rate that you have configured in Localization Workspace. For more information, see [Identify the FX Currency field and its display parameters](../../currency/task/fx-currency-display-parameters.md).
